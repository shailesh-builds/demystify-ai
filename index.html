How an LLM is built — the full story

Six stages, every term connected. Step through in order.

Stage 1 — the raw ingredients

Before anything called "AI" exists, there is just text. Enormous amounts of it — scraped from the internet, books, code, Wikipedia, scientific papers. GPT-3 alone was trained on ~45TB of text.

~15T
tokens of training data
45 TB
raw text (GPT-3)
0
model exists yet
The model doesn't "read" this data like you read a book. It sees it as a giant prediction task — given the words so far, what comes next?

Stage 2 — tokens: the atomic unit

The model can't read letters directly. A tokeniser splits every piece of text into tokens — chunks the model processes as numbers. Roughly ¾ of a word per token on average.

Sentence: "The Renaissance changed everything"

The Ren aissance changed everything

5 tokens · "Renaissance" splits because it's rare · each token becomes a number, then an embedding vector

Each token ID maps to an embedding — a list of ~1,000–12,000 numbers capturing its meaning. This is the bridge from language to maths.

Stage 3 — pre-training: the big expensive one

The model reads tokens, predicts the next one, checks if it was right, adjusts its parameters slightly, and repeats — trillions of times. This is where knowledge is absorbed.

Months
duration
$100M+
typical cost
1000s
GPUs running 24/7
At the end of pre-training: the model is extraordinarily capable but has no personality and no idea how to behave as an assistant. Ask it something and it might just continue your sentence.

Stage 4 — fine-tuning: teaching it to behave

Using a much smaller curated dataset of human-written Q&A pairs (~10K–100K examples), the model is trained to be a helpful assistant. Same knowledge, new behaviour.

Before fine-tuning: "What is 2+2?" → "What is 2+2? This question appears in many primary school curricula..."
After fine-tuning: "What is 2+2?" → "2+2 = 4."

Stage 5 — RLHF: the human feedback loop

Reinforcement Learning from Human Feedback. The model generates multiple answers. Humans rank them. A reward model learns what "good" looks like. That signal is used to nudge the main model toward higher-rated outputs.

Real example — "How do I get rid of a headache?"

★★★★★
"Try drinking water, resting in a dark room, or taking paracetamol. See a doctor if it persists."
★★★☆☆
"Headaches have many causes. Common remedies include staying hydrated and avoiding bright light..."
★☆☆☆☆
"Here is a comprehensive list of 47 things that can cause headaches: 1. Tension headaches caused by..."
This is why Claude says "I don't know" instead of making things up. RLHF trained it to prefer honest uncertainty over confident hallucination.

Stage 6 — inference: the moment you type

The model is trained and frozen. Every time you send a message, here is what happens in milliseconds:

  1. Tokenise — your message splits into token IDs
  2. Embed — each ID becomes a vector of numbers
  3. Attend — each token looks at all others, weighting relevance
  4. Transform — vectors flow through 96+ layers of parameters
  5. Predict — output probabilities for every next token; pick one; repeat
Every word you see me write is one token predicted at a time. There is no "full answer" pre-computed — it generates left to right, using everything from pre-training + fine-tuning + RLHF.

"What is the capital of France?"

Follow one question through every stage of LLM inference — tokens, embeddings, attention, and output.

Step 1 — tokenisation

Your question is split into tokens. Each becomes a number the model can process.

What is the capital of France ?

7 tokens · "France" = token ID 14881 · "capital" = token ID 6864 · and so on

Notice the space before each word is part of the token. The model learns that " France" (with a space) is different from "France" at the start of a sentence.

Step 2 — embeddings: meaning as coordinates

Each token ID maps to a vector — a list of numbers placing the word in a high-dimensional space where similar meanings cluster together.

France
Germany
Italy
Paris
Berlin
Rome
Apple
Banana
Mango

Click any dot to explore the embedding space

Countries cluster near countries. Capitals cluster near capitals. France and Paris are semantically closer than France and Apple — the model learned this from reading billions of sentences.

Step 3 — attention: what the model focuses on

Attention heads let each token look at every other token. The thickness shows how much "France" attends to each other word when predicting the answer.

What is the capital of France ?

Border thickness = attention weight. "France" and "capital" are the hot tokens — they carry the most signal for generating the answer.

This is what makes transformers powerful. Unlike older models, attention lets every token see every other token simultaneously — not just left-to-right.

Step 4 — the output: one token at a time

The model doesn't output "Paris" all at once. It outputs probabilities for every possible next token, picks the highest, appends it, and repeats.

→ "The capital of France is Paris."
For factual questions like this, temperature barely matters — Paris wins by such a massive margin. This is different for open-ended questions like "write a paragraph on the Renaissance" — which is a full access example.
